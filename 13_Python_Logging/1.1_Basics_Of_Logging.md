# Python Logging Tutorial

## 1. Why Logging Exists

Most beginners do this:

```python
print("User logged in")
print("Error occurred")
```

Problems:

* Cannot control output level
* Hard to debug large systems
* Cannot write logs to files
* Cannot filter messages

Logging solves this.

---

## 2. Basic Logging

Python has a built-in module:

```python
import logging
```

Example:

```python
import logging

logging.warning("This is a warning")
```

Output:

```
WARNING:root:This is a warning
```

Structure:

```
LEVEL : LOGGER_NAME : MESSAGE
```

---

# 3. Logging Levels

Logging has severity levels.

| Level    | Purpose                     |
| -------- | --------------------------- |
| DEBUG    | Detailed info for debugging |
| INFO     | General program events      |
| WARNING  | Something unexpected        |
| ERROR    | A failure happened          |
| CRITICAL | Program may stop            |

Example:

```python
import logging

logging.debug("Debug message")
logging.info("Info message")
logging.warning("Warning message")
logging.error("Error message")
logging.critical("Critical message")
```

But you will only see:

```
WARNING
ERROR
CRITICAL
```

Because **default level = WARNING**.

---

## 4. Setting Logging Level

To see all logs:

```python
import logging

logging.basicConfig(level=logging.DEBUG)

logging.debug("Debug message")
logging.info("Info message")
logging.warning("Warning message")
```

Output:

```
DEBUG:root:Debug message
INFO:root:Info message
WARNING:root:Warning message
```

---

## 5. Logging to a File

Instead of printing logs to console, store them.

```python
import logging

logging.basicConfig(
    filename="app.log",
    level=logging.DEBUG
)

logging.info("Application started")
logging.error("Something went wrong")
```

Logs will be saved in:

```
app.log
```

Example file content:

```
INFO:root:Application started
ERROR:root:Something went wrong
```

---

## 6. Custom Log Format

You can customize log structure.

Example:

```python
import logging

logging.basicConfig(
    level=logging.DEBUG,
    format="%(asctime)s - %(levelname)s - %(message)s"
)

logging.info("Server started")
```

Output:

```
2026-03-04 10:21:55 - INFO - Server started
```

Common formatting fields:

| Field         | Meaning     |
| ------------- | ----------- |
| %(asctime)s   | Timestamp   |
| %(levelname)s | Log level   |
| %(message)s   | Log message |
| %(name)s      | Logger name |
| %(filename)s  | File name   |
| %(lineno)d    | Line number |

Example advanced format:

```python
format="%(asctime)s | %(levelname)s | %(filename)s:%(lineno)d | %(message)s"
```

## 7. Creating Custom Logger

Instead of using the root logger, create your own.

```python
import logging

logger = logging.getLogger("myapp")

logger.warning("Something happened")
```

Output:

```
WARNING:myapp:Something happened
```

---

## 8. Using Handlers

Handlers decide **where logs go**.

Examples:

* Console
* File
* Remote server

Example:

```python
import logging

logger = logging.getLogger("app")
logger.setLevel(logging.DEBUG)

handler = logging.FileHandler("app.log")

logger.addHandler(handler)

logger.info("Hello")
```

Now logs go to **file only**.

---

## 9. Multiple Handlers (Console + File)

Very common in real applications.

```python
import logging

logger = logging.getLogger("app")
logger.setLevel(logging.DEBUG)

console = logging.StreamHandler()
file = logging.FileHandler("app.log")

logger.addHandler(console)
logger.addHandler(file)

logger.info("Server started")
```

Now logs appear:

* terminal
* log file

## 10. Logging Exceptions

One of the **most useful features**.

Instead of:

```python
try:
    1 / 0
except Exception as e:
    print(e)
```

Use logging:

```python
import logging

try:
    1 / 0
except Exception:
    logging.exception("Division failed")
```

Output includes **full stack trace**.

---

## 11. Best Practices

Use this pattern in real projects:

```python
import logging

logger = logging.getLogger(__name__)

logger.debug("Debug message")
logger.info("User logged in")
logger.warning("Unexpected input")
logger.error("Database failed")
```

Why `__name__`?

Because every file gets its own logger.

Example:

```
auth.py
database.py
server.py
```

Each module logs separately.

---


## 13. When to Use Each Level

Good rule:

```
DEBUG → developer debugging
INFO → normal operations
WARNING → unexpected but not fatal
ERROR → something failed
CRITICAL → system crash risk
```

Example:

```python
logger.debug("Loading model")
logger.info("User logged in")
logger.warning("API latency high")
logger.error("Database connection failed")
logger.critical("System shutting down")
```
