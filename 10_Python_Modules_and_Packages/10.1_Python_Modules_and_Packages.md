# Modules and Packages
## What is a MODULE ?

A **module** is just **one `.py` file**.

That’s it.

```python
math.py        # module
utils.py       # module

```

If it’s a Python file, it’s a module.

---

## Why modules exist 

Without modules:

- your code becomes a mess
- names collide
- reuse is impossible
- scaling is painful

Modules solve:

- **organization**
- **reuse**
- **readability**

---

##  Creating your own module

### utils.py

```python
def add(a, b):
    return a + b

def sub(a, b):
    return a - b

```

### main.py

```python
import utils

print(utils.add(2, 3))

```

That’s it.

You just created and used a module.

---

##  Import Variations

###  `import module`

```python
import math
math.sqrt(16)

```



---

###  `import module as alias`

```python
import numpy as np
np.array([1,2,3])

```

---

###  `from module import name`

```python
from math import sqrt
sqrt(16)

```

---

###  `from module import *` (Not Preferred)

```python
from math import *

```
Why not preferred?

1) namespace pollution

2)  debugging nightmare

---

##  What is a PACKAGE 

A **package** is a **folder of modules**.

```
project/
│
├── main.py
└── mypackage/
    ├── __init__.py
    ├── math_utils.py
    └── string_utils.py

```

Folder + Python files = package.

---

##  `__init__.py`

Truth:

- It marks the folder as a package
- It runs when the package is imported

Minimal usage:

```python
# mypackage/__init__.py

```

Even empty is fine.

Advanced stuff exists — **ignore for now**.

---

## Importing from packages

```python
from mypackage import math_utils
math_utils.add(1, 2)

```

Or:

```python
from mypackage.math_utils import add
add(1, 2)

```

Both are valid.

---

##  Absolute vs Relative imports 

### Absolute (preferred)

```python
from mypackage.math_utils import add

```

### Relative (inside package only)

```python
from .math_utils import add

```

Rule:

- Use **absolute** in real projects
- Use **relative** only inside packages

---

##  Built-in vs Third-party vs Your modules

### Built-in

```python
import sys, os, math, time

```

### Third-party

```python
import numpy
import requests

```

### Your code

```python
import utils
import mypackage.math_utils

```

They’re imported the **same way**.

Python doesn’t care.

---

## How Python finds modules 

Python searches in this order:

1. Current directory
2. Installed packages
3. `sys.path`

That’s all you need.

```python
import sys
print(sys.path)

```

---

##  `if __name__ == "__main__"` (IMPORTANT)

```python
def main():
    print("running")

if __name__ == "__main__":
    main()

```

Meaning:

- Runs only if file is executed directly
- Not when imported

This is **mandatory knowledge**.

Used everywhere.
