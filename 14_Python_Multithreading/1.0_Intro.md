### Program

A **program** is a set of instructions written in a language that a computer can understand.
It is designed to perform a specific task, but by itself it is just a file stored on disk.

Example:
A Python script or a compiled application like a browser.

---

### Process

A **process** is an instance of a program that is currently being executed.

When a program starts running, the operating system creates a process for it.
The process needs several resources from the operating system in order to run properly, such as memory, CPU time, and access to storage or devices.

Because of this, processes always work together with the **operating system**, which manages and allocates these resources.

---

### Resources Used by a Process

Each process has its own memory space and resources. These include:

* **Code Segment**
  Contains the program instructions that are executed.

* **Data Segment**
  Stores global and static variables.

* **Heap Memory**
  Used for dynamically allocated memory during runtime.

* **Stack**
  Stores function calls, local variables, and execution frames.

* **Registers**
  Small CPU storage that keeps the current execution state such as the program counter and temporary values.

Because every process has its own memory space, **processes are isolated from each other**.
One process cannot directly interfere with the memory of another process.

You can observe running processes on your computer:

```
Windows + R
→ type: taskmgr
→ open Task Manager
```

This shows all processes currently running on the system.

---

### Thread

A **thread** is the smallest unit of execution within a process.

A process can contain one or more threads. Threads allow a program to perform multiple tasks concurrently.

Each thread has its own:

* Stack
* CPU registers

But threads **share the following resources of the process**:

* Code segment
* Data segment
* Heap memory

Because of shared memory, threads can communicate easily, but this can also lead to issues such as **race conditions** if multiple threads modify shared data at the same time.

---

### Types of Processes

A process can be:

**Single-threaded process**
Only one thread executes instructions.

**Multi-threaded process**
Multiple threads run within the same process, allowing concurrent execution of tasks.

---

### Summary

* A **program** is a set of instructions stored on disk.
* A **process** is a running instance of that program.
* Each process has its own memory and resources, so processes are isolated.
* A **thread** is the execution unit inside a process.
* Threads share memory within a process but have their own stack and registers.
