

# Python Multithreading Tutorial

## 1. What is Multithreading?

Normally a Python program runs **one instruction at a time**.

Example:

```python
task1()
task2()
task3()
```

Execution:

```
task1 → finish
task2 → finish
task3 → finish
```

Multithreading allows **tasks to run concurrently**.

```
task1
task2   → running together
task3
```

Important: In Python, multithreading mostly helps with **I/O operations**, not CPU-heavy work.

---

## 2. Why Multithreading is Useful

Good use cases:

* Downloading files
* API calls
* Reading/writing files
* Waiting for database responses
* Network requests

Bad use cases:

* Heavy CPU computation
* Machine learning training
* Data processing loops

For CPU-heavy work → **multiprocessing** is better.

---

## 3. Basic Thread Creation

Python module:

```python
import threading
```

Simple example:

```python
import threading

def task():
    print("Task running")

t = threading.Thread(target=task)

t.start()
t.join()
```

Explanation:

* `Thread()` → creates a thread
* `start()` → starts execution
* `join()` → wait until thread finishes

---

## 4. Multiple Threads Example

```python
import threading

def task(name):
    print(f"Task {name} running")

t1 = threading.Thread(target=task, args=("A",))
t2 = threading.Thread(target=task, args=("B",))

t1.start()
t2.start()

t1.join()
t2.join()
```

Output order may vary:

```
Task A running
Task B running
```

Thread scheduling is **non-deterministic**.

---

## 5. Real Example — Parallel Tasks

Example with delay:

```python
import threading
import time

def work():
    print("Starting work")
    time.sleep(2)
    print("Finished work")

t1 = threading.Thread(target=work)
t2 = threading.Thread(target=work)

t1.start()
t2.start()

t1.join()
t2.join()
```

Without threading:

```
2s + 2s = 4s
```

With threading:

```
~2s
```

Because tasks overlap.

## 6. Thread with Class

You can subclass `Thread`.

```python
import threading

class MyThread(threading.Thread):

    def run(self):
        print("Thread running")

t = MyThread()

t.start()
t.join()
```

Usually people prefer the **target function approach**.

## 7. Threading Problem — Race Condition

Threads sharing data can cause issues.

Example:

```python
import threading

counter = 0

def increment():
    global counter
    for _ in range(100000):
        counter += 1

t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)

t1.start()
t2.start()

t1.join()
t2.join()

print(counter)
```

Expected:

```
200000
```

But you might get:

```
183421
197832
```

This is a **race condition**.

Multiple threads modify the same variable simultaneously.

---

## 8. Fixing Race Condition with Lock

Use **thread locks**.

```python
import threading

counter = 0
lock = threading.Lock()

def increment():
    global counter
    for _ in range(100000):

        lock.acquire()

        counter += 1

        lock.release()

t1 = threading.Thread(target=increment)
t2 = threading.Thread(target=increment)

t1.start()
t2.start()

t1.join()
t2.join()

print(counter)
```

Now result is stable.

Better style:

```python
with lock:
    counter += 1
```

Cleaner and safer.

## 9. Thread Pool (Very Important)

Creating many threads manually is inefficient.

Use **ThreadPoolExecutor**.

```python
from concurrent.futures import ThreadPoolExecutor

def task(n):
    return n * n

with ThreadPoolExecutor(max_workers=4) as executor:

    results = executor.map(task, range(10))

print(list(results))
```

This automatically:

* creates threads
* distributes tasks
* manages lifecycle

---

## 10. Real World Example — Downloading URLs

```python
import requests
from concurrent.futures import ThreadPoolExecutor

urls = [
    "https://example.com",
    "https://google.com",
    "https://github.com"
]

def download(url):
    r = requests.get(url)
    return len(r.text)

with ThreadPoolExecutor(max_workers=3) as executor:

    results = executor.map(download, urls)

print(list(results))
```

This downloads **multiple websites simultaneously**.

## 11. The GIL (Important Reality)

Python has something called the **Global Interpreter Lock (GIL)**.

Meaning:

```
Only ONE thread executes Python bytecode at a time
```

So CPU-heavy tasks do **not scale with threads**.

Example:

```
Threads → good for IO
Multiprocessing → good for CPU
```

This is why data science and ML often use:

* multiprocessing
* C libraries
* NumPy (runs outside GIL)
